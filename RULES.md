## 1. СТРУКТУРА ПРОЕКТА

```
dead-tactics/
├── index.html        ← только HTML-разметка, никакой логики
├── RULES.md          ← этот файл
├── css/
│   └── styles.css    ← всё визуальное, CSS-переменные, анимации
└── js/
    ├── config.js     ← константы (COLS, ROWS, параметры юнитов, WEAPONS)
    ├── state.js      ← глобальный стейт игры (units, phase, selected)
    ├── units.js      ← фабрики юнитов (mkPlayer, mkZombie)
    ├── helpers.js    ← чистые утилиты (manhattan, unitAt, reachable)
    ├── ui.js         ← лог, сайдбар, оверлей (только UI, без логики)
    ├── render.js     ← render(), buildGrid(), cell() — чистая отрисовка
    ├── ai.js         ← AI зомби (runZombies, zombieMove, zombieAttack)
    ├── audio.js      ← звуковой движок (Web Audio API, процедурные звуки)
    └── game.js       ← игровая логика, обработчики кликов, старт игры    ├── screens.js    ← навигация между экранами (transition logic)
    ├── stats.js      ← глобальный накопитель статистики боя```

**Правило:** каждый файл отвечает ровно за одну область. Не смешивай логику и UI.

## 4. СТЕЙТ ИГРЫ (state.js)

```
units[]       — массив всех юнитов (живых и мёртвых)
selected      — текущий выбранный юнит игрока (null если нет)
phase         — 'placement' | 'player' | 'zombie' | 'over'
placedCount   — сколько юнитов расставлено
turnNum       — номер текущего хода
turnsSurvived — сколько ходов выжило (считается, когда ход передаётся обратно игроку)
highlights    — { move: Set<"x,y">, attack: Set<"x,y"> }
```

**Никогда не создавай отдельный стейт в других файлах.** Всё через эти переменные.

## 5. ПАРАМЕТРЫ ЮНИТОВ (config.js)

| Параметр        | Выживший | Зомби |
|----------------|---------|-------|
| hp / maxHp     | 5       | 3     |
| moveRange      | 5       | 3     |
| atkRange       | 4       | 1     |
| weapon         | pistol  | —     |
| atkDmg         | —       | 1     |
| poison (укус)  | —       | +1HP/ход |

Игрок может размещать одновременно не более `leadership` юнитов; это значение хранится в `gameData.player.leadership`. В начале оно равно 1.
Оружие игрока: WEAPONS = { pistol: { baseDmg: 1, critDmg: 2, critChance: 0.10 } }

Менять баланс → только в config.js, не хардкодить числа в логике.

## 13. УНИКАЛЬНОСТЬ ID В DOM

### 13.1 Главное правило
**id должны быть УНИКАЛЬНЫ во всём DOM** - не только в одном файле.

### 13.2 Почему это важно
- `document.getElementById()` ищет во всём document
- SVG pattern с id="grid" конфликтует с div id="grid"
- Это вызывает необъяснимые баги (размеры 0×0, неправильные ссылки)

### 13.3 Запрещённые имена (без префикса)
- grid, box, item, title, container, panel, map, cell
- Все generic имена должны иметь префикс экрана

### 13.4 Правило именования
- HTML id: `<div id="battle-grid">`
- SVG pattern: `<pattern id="map-grid-pattern">`
- Все динамические id в JS должны проверяться через grep

### 13.5 Проверка перед созданием
Перед добавлением нового id:
1. grep по проекту на этот id
2. Если есть совпадение - добавить префикс

### 13.6 CSS классы vs ID
- Для повторяющихся элементов - классы
- id только для уникальных элементов (1 на странице)
