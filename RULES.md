# ☣ DEAD TACTICS — RULES FOR AI AGENT

> Этот файл читает AI-агент (Cline/Gemini) перед каждым изменением.
> Подключи его в системный промпт Cline: Settings → Custom Instructions → вставь содержимое.

---

## 1. СТРУКТУРА ПРОЕКТА

```
dead-tactics/
├── index.html        ← только HTML-разметка, никакой логики
├── RULES.md          ← этот файл
├── css/
│   └── styles.css    ← всё визуальное, CSS-переменные, анимации
└── js/
    ├── config.js     ← константы (COLS, ROWS, параметры юнитов, WEAPONS)
    ├── state.js      ← глобальный стейт игры (units, phase, selected)
    ├── units.js      ← фабрики юнитов (mkPlayer, mkZombie)
    ├── helpers.js    ← чистые утилиты (manhattan, unitAt, reachable)
    ├── ui.js         ← лог, сайдбар, оверлей (только UI, без логики)
    ├── render.js     ← render(), buildGrid(), cell() — чистая отрисовка
    ├── ai.js         ← AI зомби (runZombies, zombieMove, zombieAttack)
    └── game.js       ← игровая логика, обработчики кликов, старт игры
```

**Правило:** каждый файл отвечает ровно за одну область. Не смешивай логику и UI.

---

## 2. ЗАПРЕЩЕНО

- ❌ Не удаляй существующие функции — только добавляй или точечно правь
- ❌ Не переписывай файл целиком без явного запроса
- ❌ Не меняй сигнатуры функций (имя, аргументы) без обновления всех вызовов
- ❌ Не добавляй inline-стили в JS — только классы через CSS
- ❌ Не вставляй игровую логику в render.js или ui.js
- ❌ Не дублируй константы — всё в config.js
- ❌ Не используй localStorage или sessionStorage
- ❌ Не добавляй зависимости (npm, CDN-либы) без явного запроса

---

## 3. ОБЯЗАТЕЛЬНО

- ✅ Перед изменением прочитай затронутый файл целиком
- ✅ Правь только то, что явно указано в задаче
- ✅ После изменения проверь — не сломал ли вызовы в других файлах
- ✅ Новый тип юнита → добавь фабрику в units.js + параметры в config.js
- ✅ Новый экран/оверлей → только в ui.js
- ✅ Новая механика → только в game.js (+ helpers.js если нужна утилита)
- ✅ CSS-переменные (цвета, размеры) только в :root в styles.css

---

## 4. СТЕЙТ ИГРЫ (state.js)

```
units[]       — массив всех юнитов (живых и мёртвых)
selected      — текущий выбранный юнит игрока (null если нет)
phase         — 'placement' | 'player' | 'zombie' | 'over'
placedCount   — сколько юнитов расставлено
turnNum       — номер текущего хода
highlights    — { move: Set<"x,y">, attack: Set<"x,y"> }
```

**Никогда не создавай отдельный стейт в других файлах.** Всё через эти переменные.

## 5. ПАРАМЕТРЫ ЮНИТОВ (config.js)

| Параметр        | Выживший | Зомби |
|----------------|---------|-------|
| hp / maxHp     | 5       | 3     |
| moveRange      | 5       | 3     |
| atkRange       | 4       | 1     |
| weapon         | pistol  | —     |
| atkDmg         | —       | 1     |
| poison (укус)  | —       | +1HP/ход |

Оружие игрока: WEAPONS = { pistol: { baseDmg: 1, critDmg: 2, critChance: 0.10 } }

Менять баланс → только в config.js, не хардкодить числа в логике.

## 6. ИГРОВОЙ ЦИКЛ

```
startGame()
  └── phase = 'placement'
        └── игрок расставляет 2 юнита
              └── spawnZombies() → phase = 'player'
                    └── handlePlayer() / btn-end-turn
                          └── phase = 'zombie'
                                └── runZombies() → startPlayerTurn()
                                      └── phase = 'player' (или 'over')
```

## 7. СТИЛЬ КОДА

- Язык: JavaScript (vanilla, ES6+), без TypeScript
- Именование: camelCase для функций и переменных, UPPER_CASE для констант
- Комментарии: на русском, кратко, только там где неочевидно
- Функции: максимум 30 строк. Длиннее → разбить
- Отступы: 2 пробела
- Кавычки: одинарные '

## 8. ДОБАВЛЕНИЕ НОВОГО КОНТЕНТА

### Новый тип зомби:
1. config.js → добавь ZOMBIE_TYPES.newType = { hp, moveRange, ... }
2. units.js → mkNewZombie(x, y)
3. ai.js → если нужно особое поведение — новая функция zombieNewBehavior()
4. game.js → если нужна спец-механика

### Новый экран (магазин, карта):
1. ui.js → showShop(), hideShop()
2. index.html → добавь overlay-div
3. styles.css → стили для нового экрана
4. game.js → логика переходов между экранами

### Новый тип урона/эффекта:
1. state.js → добавь поле в объект юнита
2. game.js → логика применения
3. ui.js → визуальное отображение в сайдбаре
