# GAMEPLAY — Dead Tactics

---

## 1. Юниты — общие правила

### Флаги состояния

Общие для всех юнитов:
- `u.alive` — жив или мёртв
- `u.moving` — идёт по клеткам прямо сейчас
- `u.attacking` — выполняет атаку прямо сейчас
- `u.dying` — воспроизводится анимация смерти (переходное состояние)
- `u.moved` — уже переместился в этот ход
- `u.attacked` — уже атаковал в этот ход

Только survivor:
- `u.reloading` — идёт перезарядка
- `u.poisonFlash` — временный флаг для анимации яда (~300ms)
- `u.dyingAnim` — воспроизводится анимация падения (переходное состояние)
- `u.effects` — активные эффекты `{ poison: { duration, stacks } }`
- `u.shotsFired` — счётчик выстрелов для перезарядки
- `u.inventory` — расходники `{ antidote: 2, grenade: 1 }`

Только zombie:
- `u.damagedFlash` — получил обычный урон (временный)
- `u.critFlash` — получил критический урон (временный)
- `u.raged` — вошёл в состояние ярости (1 HP остался)

### Боевые правила

- Юнит не может переместиться дважды за один ход
- Атака и перемещение возможны в любом порядке
- По умолчанию атака один раз за ход. Исключение: оружие с параметром `shots > 1` (например автомат)
- Урон всегда через `calcDamage(weaponId)` — никогда не хардкодить числа
- Использование предмета тратит действие атаки (`u.attacked = true`)

### Смерть юнита

Survivor: убирается с поля полностью при `u.alive = false`.

Zombie: остаётся как труп до конца боя. Показывается только спрайт `killed`. Без HP-бара, без тултипа, без обработки кликов, без обработки состояний.

## 2. Survivor

**Параметры по умолчанию:** HP 5, перемещение 4 клетки, атака через оружие.

**Оружие по умолчанию:** пистолет — дальность 4, урон 1 (крит 2, шанс 10%).

Survivor не атакует сам — атака определяется параметрами оружия из `ITEMS` в `game-config.js`. Разное оружие — разные `baseDmg`, `critDmg`, `critChance`, `atkRange`, `shots`.

**Перезарядка:** после каждых 3 выстрелов подряд `u.reloading = true`. Анимация reload. После завершения флаг снимается.

**Предметы:** хранятся в `u.inventory[itemId] = количество`. Покупаются в магазине через `buyItem()` в `data.js`.

---

## 3. Zombie

**Параметры по умолчанию:** HP 3, перемещение 3 клетки, укус дальностью 1.

**Путь:** выбирается автоматически по манхэттенскому расстоянию до ближайшего survivor. Логика в `ai.js`.

**Укус:** урон 1 HP + накладывает яд на survivor. Яд не стакается — если уже есть, повторно не накладывается. Броня с `blockPoison: true` блокирует яд.

**Ярость:** когда HP становится 1 — `u.raged = true`.
- В idle воспроизводится loop-анимация `raged` вместо `idle`
- В свой ход атакует дважды (две анимации `attack` с паузой)
- Урон и яд не меняются

## 4. Эффекты

Вся система в `src/js/core/effects.js` .

**API:**
- `addEffect(unit, effectId, duration)` — наложить эффект
- `removeEffect(unit, effectId)` — снять эффект
- `hasEffect(unit, effectId)` — проверить наличие
- `processEffectsOnTurnStart(unit)` — обработать в начале хода
- `getEffectIcons(unit)` — иконки для отображения

**Хранение:** `u.effects = { poison: { id, duration, stacks, applied } }`.
`duration: null` = бесконечно. `stacks` — зарезервировано, пока не влияет на логику.

**Когда срабатывает:** зависит от параметра `onTurnStart` в конфиге эффекта. Яд срабатывает в начале каждого хода игрока в `startPlayerTurn()`. Другие эффекты могут иметь другую логику — определяется в `EFFECT_HANDLERS`.

**Яд:** накладывается укусом зомби, действует до конца боя или до снятия предметом. Урон 1 HP в начале каждого хода survivor. Анимация — через временный флаг `u.poisonFlash`, не через `u.effects.poison`.

**Нельзя:** менять `u.effects` напрямую, вызывать логику эффектов минуя `processEffectsOnTurnStart`, путать `u.poisonFlash` (анимация) с `u.effects.poison` (игровой эффект).

### Добавление нового эффекта

1. Добавить конфиг в `EFFECTS` в `effects.js`
2. Если нужен новый тип обработки — добавить функцию в `EFFECT_HANDLERS`
3. Добавить логику наложения в нужное место (`ai.js`, `game.js`)
4. Если есть визуальная анимация — добавить флаг в `units.js` и состояние в `battle.js`
5. Если блокируется снаряжением — добавить параметр в `ITEMS` и проверку перед `addEffect`

## 5. Магазин

Предметы описаны в `ITEMS` в `game-config.js`. Типы: `weapon`, `armor`, `boots`, `consumable`.

**Покупка:** через `buyItem(unitId, itemId)` в `data.js`.
- Оружие/броня/обувь записываются в `unit.equipment[type] = itemId`
- Расходники записываются в `unit.inventory[itemId]++`

**Категории в UI:**
- "Основное оружие" — `weapon`
- "Защита и лечение" — `armor` + `consumable`
- "Аксессуары" — `boots`

Тип `consumable` — внутренняя классификация, пользователю не показывается как отдельная категория.

### Граната

Параметры в `ITEMS.grenade`: `throwRange` (дальность броска), `splashRange` (радиус взрыва), `damage` (урон).

**Базовая механика:** выбрать гранату из инвентаря → `activateGrenade(u)` подсвечивает зону броска (оранжевые клетки) → клик на клетку → `doGrenade()` наносит урон всем зомби в `splashRange` от точки попадания → `u.attacked = true`, `u.inventory.grenade--`.

**Режим гранаты:** При активации гранаты сохраняется `state.grenadeAttackerId = u.id`. Этот ID проверяется при каждом клике чтобы предотвратить ошибки.

**Friendly fire:** Граната может попасть в союзника, если он находится в радиусе `splashRange`. Однако союзник который бросил гранату — не получает урон от своей гранаты.

#### Сценарии броска гранаты

| Клик | Зона | Поведение |
|------|------|-----------|
| Пустая клетка | throw (оранжевая) | ✅ Бросок |
| Зомби | throw (оранжевая) | ✅ Бросок |
| Союзник (другой) | throw (оранжевая) | ❌ Бросок запрещён, показывает "[ЦЕЛЬ ЗАБЛОКИРОВАНА]" в тултипе |
| Пустая клетка | НЕ throw | ❌ Отмена гранаты |
| Зомби | НЕ throw | ❌ Отмена гранаты + выбрать зомби (инфа в сайдбар) |
| Союзник (тот же) | throw | ❌ Отмена гранаты (НЕЛЬЗЯ на себя) |
| Союзник (тот же) | НЕ throw | ❌ Отмена гранаты + снять выделение |
| Союзник (другой) | Любая | ❌ Отмена гранаты + выбрать другого |

#### Функции гранаты

| Функция | Файл | Назначение |
|---------|------|------------|
| `activateGrenade(u)` | `game.js` | Активирует режим гранаты, сохраняет `grenadeAttackerId` |
| `doGrenade(attacker, x, y)` | `game.js` | Выполняет бросок с анимацией и валидацией |
| `handlePlayer(c, r)` | `game.js` | Обрабатывает все клики с учётом режима гранаты |
| `state.grenadeAttackerId` | `state.js` | ID survivor который активировал гранату |
| `state.getGrenadeAttackerId()` | `state.js` | Геттер для получения ID |
| `state.setGrenadeAttackerId(id)` | `state.js` | Сеттер для установки ID |

#### Валидация

В `doGrenade()` выполняются проверки:
1. `attacker.kind === 'survivor'` — атакующий должен быть survivor
2. `attacker.id === grenadeAttackerId` — должен быть тот же survivor который активировал гранату
3. `attacker.inventory.grenade > 0` — должна быть граната в инвентаре

## 6. Бой — фазы и порядок хода

Фазы: `placement` → `player` → `zombie` → `over`

**Расстановка:** игрок кликает на зелёные клетки в `PLACE_COLS`. После расстановки всех юнитов — спавн зомби, фаза `player`.

**Ход игрока:** клик на юнита → подсветка зон. Клик на синюю зону → движение. Клик на красную зону (зомби) → атака. Клик на оранжевую зону → бросок гранаты. Кнопка "Завершить ход" → фаза `zombie`.

Зоны подсветки хранятся в `state.highlights`:
- `highlights.move` — синие клетки, доступные для перемещения
- `highlights.attack` — красные клетки с зомби в радиусе атаки
- `highlights.throw` — оранжевые клетки в радиусе броска гранаты

**Ход зомби:** `runZombies()` в `ai.js`. Каждый зомби двигается и/или атакует. После всех зомби → `startPlayerTurn()`.

**Конец боя:** все зомби мертвы → победа. Все survivor мертвы → поражение.

## 7. Атака и урон

### Централизованная функция takeDamage()

**ВАЖНО:** ВСЕГДА используйте `takeDamage()` для нанесения урона! НЕЛЬЗЯ менять HP напрямую (`u.hp -= X`).

```javascript
takeDamage(target, amount, source)
// target: юнит которому наносим урон
// amount: количество урона
// source: 'player' | 'zombie' | 'grenade' | 'effect'
// returns: { died: boolean }
```

**Что делает takeDamage():**
1. Вычитает HP: `target.hp -= amount`
2. Записывает урон в статистику
3. Ставит флаг `damagedFlash = true` (анимация повреждения)
4. Для source='player' и amount >= 2 ставит `critFlash = true`
5. При смерти (hp <= 0):
   - Для survivor: ставит `dyingAnim = true` (анимация падения)
   - Для zombie: ставит `dying = true` (анимация смерти)
6. Проверяет ярость зомби: если hp === 1 → `raged = true`

### Основные функции

| Функция | Файл | Назначение |
|---------|------|------------|
| `takeDamage(target, amount, source)` | `game.js` | **Централизованное нанесение урона** |
| `doAttack(attacker, target)` | `game.js` | Логика атаки выжившего |
| `calcDamage(weaponId)` | `helpers.js` | Расчёт урона с учётом крита |
| `getEffectiveStat(unit, stat)` | `helpers.js` | Получение статов с учётом снаряжения |
| `zombieAttack(z, target)` | `ai.js` | Атака зомби (укус) |

### Процесс атаки выжившего

1. **Проверка:** юнит может атаковать (`!u.attacked`)
2. **Оружие:** определяется через `getEffectiveStat(attacker, 'weapon')`
3. **Урон:** рассчитывается через `calcDamage(weaponId)` → массив [dmg1, dmg2]
4. **Нанесение:** `takeDamage(target, damage, 'player')` — ВСЕГДА через takeDamage!
5. **Анимация и смерть:** обрабатываются внутри takeDamage()

### Визуализация урона

После изменения HP вызывается `render()`:
- `render()` → `syncUnitsWithDOM()` → `updateUnitVisuals()`
- Функция `updateUnitVisuals()`:
  - Обновляет ширину и класс цвета HP bar
  - Удаляет HP bar при смерти юнита
  - Не пересоздаёт DOM-элемент — избегает визуального мигания

### Важно

- **НЕЛЬЗЯ** менять `u.hp` напрямую — только через `takeDamage()`
- Урон всегда через `calcDamage()` — **никогда не хардкодить числа**
- HP зомби отображается в sidebar при клике на зомби
- HP bar обновляется автоматически через `updateUnitVisuals()`
